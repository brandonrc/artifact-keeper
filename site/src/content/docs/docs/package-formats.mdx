# Package Formats

Artifact Keeper supports **45+ package formats** out of the box. Every format is included at no extra cost — no feature gates, no tier restrictions. Each speaks the native wire protocol that your package manager already expects.

Custom formats can be added via the [WASM plugin system](/docs/advanced/plugins/).

---

## Languages & Runtimes

### Java / JVM

| Format | Key | API Endpoint | Native Clients |
|--------|-----|--------------|----------------|
| **Maven** | `maven` | `/maven/{repo}` | Maven, Gradle, Ant+Ivy |
| **SBT** | `sbt` | `/ivy/{repo}` | sbt, Ivy |

**Maven** (`pom.xml`):
```xml
<repositories>
  <repository>
    <id>artifact-keeper</id>
    <url>https://registry.example.com/maven/my-repo</url>
  </repository>
</repositories>
```

**SBT** (`build.sbt`):
```scala
resolvers += "Artifact Keeper" at "https://registry.example.com/ivy/my-repo"
```

## Containers & OCI

| Format | Key | API Endpoint | Native Clients |
|--------|-----|--------------|----------------|
| **Docker / OCI** | `docker` | `/v2/{repo}` | Docker, Podman, Buildx, ORAS, Helm OCI, WASM OCI |

Full OCI Distribution Spec implementation. Aliases: `oci`, `podman`, `buildx`, `oras`, `wasm_oci`, `helm_oci` all route through the unified OCI handler.

**Docker**:
```bash
docker login registry.example.com
docker push registry.example.com/my-repo/myimage:latest
```

**Podman**:
```bash
podman push registry.example.com/my-repo/myimage:latest
```

**ORAS** (generic OCI artifacts):
```bash
oras push registry.example.com/my-repo/myartifact:v1 ./artifact.tar.gz
```

## System Packages

| Format | Key | API Endpoint | Native Clients |
|--------|-----|--------------|----------------|
| **Debian / APT** | `debian` | `/debian/{repo}` | apt, dpkg |
| **RPM / YUM** | `rpm` | `/rpm/{repo}` | yum, dnf, zypper |
| **Alpine / APK** | `alpine` | `/alpine/{repo}` | apk |
| **OPKG** | `opkg` | `/opkg/{repo}` | opkg (OpenWrt) |

**Debian** (`/etc/apt/sources.list.d/artifact-keeper.list`):
```
deb https://registry.example.com/debian/my-repo stable main
```

**RPM** (`/etc/yum.repos.d/artifact-keeper.repo`):
```ini
[artifact-keeper]
name=Artifact Keeper
baseurl=https://registry.example.com/rpm/my-repo
enabled=1
gpgcheck=1
```

**Alpine** (`/etc/apk/repositories`):
```
https://registry.example.com/alpine/my-repo/v3.19/main
```

**OPKG** (`/etc/opkg/customfeeds.conf`):
```
src/gz artifact-keeper https://registry.example.com/opkg/my-repo
```

:::note[Package signing]
Debian, RPM, Alpine, and Conda repositories support GPG/RSA signing for repository metadata. See [Artifact Signing](/docs/security/signing/) for setup.
:::

## Generic

| Format | Key | API Endpoint | Native Clients |
|--------|-----|--------------|----------------|
| **Generic** | `generic` | `/generic/{repo}` | curl, wget, any HTTP client |

Store any file type with custom metadata. Useful for build outputs, documentation bundles, or proprietary formats.

```bash
# Upload
curl -X PUT \
  -H "Authorization: Bearer $TOKEN" \
  --data-binary @myfile.zip \
  https://registry.example.com/generic/my-repo/path/to/myfile.zip

# Download
curl -O https://registry.example.com/generic/my-repo/path/to/myfile.zip
```

## Adding Custom Formats

Artifact Keeper can be extended with custom format handlers using the WASM plugin system:

1. Write a handler in any language that compiles to WASM (Rust, Go, C, AssemblyScript)
2. Implement the `FormatHandler` WIT interface (`parse_metadata`, `validate`, `generate_index`)
3. Install via Git URL, ZIP upload, or local path

See [WASM Plugins](/docs/advanced/plugins/) for details.
